First open source contribution

Default admin-cli client for every realm

Project: FerrisKey · Core layer · 2025

The project: FerrisKey is an open source Identity & Access Management (IAM) system, written in Rust with a hexagonal architecture. It offers a lightweight, performant alternative to proprietary IAMs: multi-tenant realms, OIDC/OAuth2 flows, MFA (TOTP), audit, webhooks, and Kubernetes deployment via Helm. The project emphasizes clear layer separation (core, api, front, operator) and focused, reviewable pull requests.

My contribution

On first startup, FerrisKey creates the master realm via an initialization routine that did not create the admin-cli client. This client is used for command-line administration (public client with direct access by credentials). It was already created automatically for all new realms, but not for the master realm.

I added creation of the admin-cli client in the initialize_application() method of the ferriskey-core crate, with the same configuration as for other realms: public: true, direct access grant: true, and idempotent behavior (create only if the client does not already exist).

Impact on the project

Now every realm — including the master realm — has a default admin-cli client. Administrators can use the same client for scripts and CLI tools across all realms, without extra manual setup. Behavior is consistent between initial bootstrap and creation of new realms.

  • Single file changed (core/src/domain/common/services.rs), single responsibility
  • Follows project conventions (idempotent pattern, logging, error handling)
  • Tests and clippy passing, PR ready for review
